## 2. Conda 환경 설정 (권장)

### 2.1 환경 생성

```bash
conda create -n em25d python=3.12
conda activate em25d
```

### 2.2 핵심 패키지 설치 (MKL 기반)

**MKL scipy가 성능의 핵심입니다.** pip 대신 conda로 설치해야 MKL BLAS/LAPACK이 연결됩니다.

```bash
# numpy + scipy (MKL 백엔드) — 반드시 conda로 설치
conda install numpy scipy "blas=*=mkl"

# MPI 병렬화
pip install mpi4py

# PARDISO direct solver (MKL 기반, 선택)
pip install pypardiso
```

### 2.3 MKL 설치 확인

```bash
python -c "
import numpy as np
np.show_config()
" 2>&1 | grep -A2 "blas:"
```

출력에 `name: mkl` 또는 `name: mkl-sdl`이 보여야 합니다.
`openblas`가 보이면 pip scipy가 우선 로드된 것이므로 아래 해결법을 참고하세요.

#### 문제 해결: pip scipy가 MKL을 덮어씀

pip로 scipy를 설치한 적이 있다면 자체 OpenBLAS를 번들하고 있어 conda MKL이 무시됩니다.

```bash
# pip scipy 제거 후 conda로 재설치
pip uninstall scipy -y
conda install scipy "blas=*=mkl"

# numpy도 pip 버전이면 호환성 문제 발생 가능
# scipy 1.17+는 numpy 2.1+ 필요
pip install "numpy>=2.1"
```

### 2.4 MKL 성능 검증

```bash
python -c "
import time, numpy as np
import scipy.linalg.lapack as lapack

n = 9150; kl = ku = 76
ab = np.random.randn(2*kl+ku+1, n) + 1j*np.random.randn(2*kl+ku+1, n)

t0 = time.perf_counter()
lub, piv, info = lapack.zgbtrf(ab, kl, ku)
t = time.perf_counter() - t0
print(f'zgbtrf: {t*1000:.0f} ms')
print('MKL' if t < 0.5 else 'OpenBLAS (느림!)')
"
```

- **MKL**: ~89ms
- **OpenBLAS**: ~2100ms (23배 느림)

## 4. 실행 방법

### 4.1 기본 검증 (직렬)

```bash
cd em25d_python

# 직렬 실행 (solver 자동 선택: banded > PARDISO > gcrotmk)
OMP_NUM_THREADS=1 MKL_NUM_THREADS=1 python scripts/verify_legacy.py --solver direct
```

### 4.2 MPI 병렬 실행 (최적)

```bash
# 핵심: 스레드를 1로 고정해야 코어 경합 방지
OMP_NUM_THREADS=1 MKL_NUM_THREADS=1 mpirun -np 20 python scripts/verify_legacy.py --solver direct
```

스레드 환경변수를 설정하지 않으면 MPI 프로세스 × BLAS 스레드로 CPU 경합이 발생하여
**4~6배 느려집니다.**

### 4.3 solver 옵션

| 옵션 | 설명 | 성능 (MKL) | 성능 (OpenBLAS) |
|------|------|-----------|----------------|
| `--solver direct` | 자동 선택 (banded→PARDISO→gcrotmk) | 최적 | 최적 |
| `--solver band` | LAPACK zgbtrf/zgbtrs 강제 | 최적 (MKL) | 느림 |
| `--solver pardiso` | MKL PARDISO 강제 | 양호 | 양호 |
| `--solver gmres` | 전처리 없는 GMRES | 느림 | 느림 |
| `--solver splu` | SuperLU (비권장) | 매우 느림 | 매우 느림 |
| `--gpu` | GPU solver (PyTorch/CuPy) | — | — |

### 4.4 GPU 실행

```bash
# PyTorch CUDA (RTX 2000 Ada 이상 권장)
pip install torch --index-url https://download.pytorch.org/whl/cu121
python scripts/verify_legacy.py --gpu

# CuPy (대안)
pip install cupy-cuda12x
python scripts/verify_legacy.py --gpu
```

GPU dense solver는 이 격자 크기(9150 DOF)에서는 CPU banded solver보다 느립니다.
더 큰 격자(DOF > 50000)에서 유리해질 수 있습니다.

```bash

```

## 7. 트러블슈팅

### "zgbtrf 실패 (info=N)"
- K 행렬이 특이(singular)합니다. 비저항 모델이 올바른지 확인하세요.

### MPI 실행 시 극심하게 느림
- `OMP_NUM_THREADS=1 MKL_NUM_THREADS=1` 환경변수를 반드시 설정하세요.
- `OPENBLAS_NUM_THREADS=1`도 추가하면 안전합니다.

### "PyPardiso currently only supports float64"
- pypardiso는 complex128을 직접 지원하지 않습니다. 코드에서 자동으로 real-split 처리합니다.

### numpy/scipy 버전 충돌
```bash
# scipy 1.17+는 numpy 2.1+ 필요
pip install "numpy>=2.1"
# 또는 호환되는 scipy 설치
conda install scipy=1.14
```

### PARDISO 없이 실행
- pypardiso 미설치 시 자동으로 ILU+gcrotmk fallback (약간 느림)
- MKL scipy만 있으면 banded solver가 최적으로 작동

### MPI 프로세스가 코어 수 초과
```bash
# --oversubscribe 옵션 추가
mpirun --oversubscribe -np 20 python scripts/verify_legacy.py --solver direct
```
